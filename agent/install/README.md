# Installing the RemoteConnect agent

The agent runs on the machine you want to remote-control. Once installed
it auto-registers with the backend, holds an outbound WebSocket open, and
accepts control commands when a technician starts a session.

> ⚠ The agent grants whoever has technician credentials full mouse,
> keyboard, screen, and file-transfer access to this machine. Only install
> on machines you control, and only point it at a backend you trust.

## Linux

Tested on Ubuntu/Debian/Fedora. Requires Python 3.12+.

```bash
git clone <repo>
cd remoteconnect/agent/install
./install-linux.sh --server-url https://your.host
```

This:
- Installs the agent under `~/.local/share/remoteconnect-agent`
- Creates a venv and installs dependencies
- Writes a user systemd unit at `~/.config/systemd/user/remoteconnect-agent.service`
- Starts it under your user account so it has access to the active desktop
  (mouse/keyboard injection via pynput needs an X/Wayland display)

Manage:
```bash
systemctl --user status remoteconnect-agent
systemctl --user restart remoteconnect-agent
journalctl --user -u remoteconnect-agent -f
```

By default the unit only runs while you are logged in. To keep it running
across logout/login cycles, install with `--enable-linger`:
```bash
./install-linux.sh --server-url https://your.host --enable-linger
```

Uninstall:
```bash
./install-linux.sh --uninstall
```

## Windows

Tested on Windows 10/11. Requires Python 3.12+ from
<https://www.python.org/downloads/> (tick "Add python.exe to PATH"
during install).

Open PowerShell **as the user whose desktop you want shared** (not as
admin — the task is registered for the current user) and run:

```powershell
cd remoteconnect\agent\install
.\install-windows.ps1 -ServerUrl https://your.host
```

This:
- Installs the agent under `%LOCALAPPDATA%\RemoteConnect`
- Creates a venv and installs dependencies
- Registers a Scheduled Task `RemoteConnectAgent` that runs **at logon**
  (not as a Windows Service — services run as LocalSystem and can't inject
  input into a user session, which the agent needs)
- Starts the task immediately

Manage:
```powershell
Get-ScheduledTask -TaskName RemoteConnectAgent | Get-ScheduledTaskInfo
Stop-ScheduledTask -TaskName RemoteConnectAgent
Start-ScheduledTask -TaskName RemoteConnectAgent
```

Uninstall:
```powershell
.\install-windows.ps1 -Uninstall
```

## Daily.co video publisher (optional)

By default the agent runs in "log-only" mode — it accepts input but
doesn't publish video. To share the screen via Daily.co, install
`daily-python` in the agent venv and set `DAILY_PUBLISHER_CMD`:

**Linux:**
```bash
~/.local/share/remoteconnect-agent/venv/bin/pip install daily-python
# Edit ~/.local/share/remoteconnect-agent/agent/.env, add:
#   DAILY_PUBLISHER_CMD=/home/USER/.local/share/remoteconnect-agent/venv/bin/python -m agent.publisher_daily
systemctl --user restart remoteconnect-agent
```

**Windows:**
```powershell
& "$env:LOCALAPPDATA\RemoteConnect\venv\Scripts\pip.exe" install daily-python
# Edit %LOCALAPPDATA%\RemoteConnect\agent\.env, add:
#   DAILY_PUBLISHER_CMD=%LOCALAPPDATA%\RemoteConnect\venv\Scripts\python.exe -m agent.publisher_daily
Restart-ScheduledTask -TaskName RemoteConnectAgent  # or Stop+Start
```

The backend must also have `DAILY_API_KEY` set, otherwise it issues mock
room URLs and the publisher has nothing real to join.

## What gets installed

```
INSTALL_DIR/
├── agent/
│   ├── agent.py                  main loop
│   ├── transfer_handlers.py
│   ├── publisher_daily.py        optional Daily SDK publisher
│   ├── … (rest of agent source)
│   ├── .env                      generated by the installer
│   └── files/                    sandboxed share dir for uploads/downloads
└── venv/                         Python virtual environment
```

The installer never writes outside `INSTALL_DIR` except for:
- Linux: the user systemd unit at `~/.config/systemd/user/`
- Windows: the Scheduled Task in `\Microsoft\Windows\Task Scheduler\`

## Troubleshooting

- **Agent registers but inputs do nothing** — pynput needs an active
  desktop session. On Linux, check `echo $DISPLAY` shows something while
  the unit runs; on Windows make sure the task is running as the
  interactive user (not LocalSystem).
- **`pip install` fails on `daily-python`** — that's a native package; on
  some Linux distros you'll need `build-essential` and `libssl-dev`. The
  agent runs fine without it; you just won't have Daily video publishing.
- **Agent reconnect-loops** — check that the backend's `/machines/`
  endpoint is reachable from this machine over HTTPS, and that the
  per-machine token in `agent/config.json` matches the row in the
  backend's database.
